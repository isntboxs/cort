# Domain Docs

This is a single-context repo.

Before exploring, engineering skills should read these if present:

- `CONTEXT.md` at the repo root
- ADRs under `docs/adr/` that touch the area being changed

If these files do not exist, proceed silently. Do not create them upfront; domain docs are created lazily when useful.

Use vocabulary from `CONTEXT.md` when it exists. If output contradicts an ADR, surface that conflict explicitly.
