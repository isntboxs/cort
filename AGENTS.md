## Skill Loading

Before substantial work:

- Skill check: run `npx @tanstack/intent@latest list`, or use skills already listed in context.
- Skill guidance: if one local skill clearly matches the task, run `npx @tanstack/intent@latest load <package>#<skill>` and follow the returned `SKILL.md`.
- Monorepos: when working across packages, run the skill check from the workspace root and prefer the local skill for the package being changed.
- Multiple matches: prefer the most specific local skill for the package or concern you are changing; load additional skills only when the task spans multiple packages or concerns.

## Project Context

- Scaffold command: `npx @tanstack/cli@latest create my-tanstack-app --agent`.
- CLI choices: React, file-router mode, no toolchain add-on, Nitro agnostic deployment adapter, no demo/example pages, no add-ons, initialize git.
- Follow-up TanStack Intent commands run: `npx @tanstack/intent@latest install`, `npx @tanstack/intent@latest list`, `npx @tanstack/intent@latest load @tanstack/start-client-core#start-core`, `npx @tanstack/intent@latest load @tanstack/react-start#react-start`, `npx @tanstack/intent@latest load @tanstack/start-client-core#start-core/deployment`.
- Package manager: Bun. The CLI installed with npm because the requested scaffold command used `npx`; this repo keeps `bun.lock` and `packageManager: bun@1.3.14`.
- Stack: TanStack Start with React, TanStack Router file routes, Vite, Nitro, Tailwind CSS as generated by the CLI.
- Partner integration: CodeRabbit is external repository tooling only. It is represented by `.coderabbit.yaml`; install/enable the CodeRabbit GitHub App for the repository to make PR reviews run.

## Environment

- No app environment variables are required for the blank starter.
- Keep secrets server-side. Client-exposed environment variables must use the Vite `VITE_` prefix only when they are intentionally public.

## Deployment

- Default adapter is Nitro agnostic.
- Build with `bun run build`.
- Start the built Node server with `node .output/server/index.mjs`.
- For Bun runtime deployment, TanStack guidance says React 19 is required and Nitro can use `nitro({ preset: 'bun' })`; only add that when deploying to Bun.

## Architecture Notes

- Preserve the generated file-router structure under `src/routes`.
- `routeTree.gen.ts` is generated by TanStack Router; do not hand-edit it.
- TanStack Start code is isomorphic by default. Use `createServerFn` or server-only files before touching secrets, databases, or private APIs.
- In `vite.config.ts`, keep TanStack Start before React when changing plugins.
